Computer Science is very different from other fields. There are specific skills that need to be developed in it that oher fields don't necessarily require.

The goal is to eventually become a <span class="bold">good programmer</span>. But what even is a good programmer? It's not someone who codes quickly or who never encounters bugs. Rather, it's somone who can come up with the best possible solution to a computation problem, and who's then able to implement it.

<div class="highlight">
This requires a grasp of computational thinking.
</div>

Students tend to think they can “bullshit” an assignment and write whatever lets them get this over with quickly. Coding is not like that. If your program doesn’t run, it just doesn’t, and there’s no “bullshitting” that.

We want to help you reach your full potential. For you to succeed, you need to get used to a new way of thinking, which you will begin to naturally become proficient in as you dive further into Computer Science.

### What Is Computational Thinking?

<div class="highlight">
<p class="block-quote">“Computational thinking \[is\] a composite set of thinking and problem-solving skills closely related to, and learned through, coding” (Grover 105)</p>
</div>

It's a grouping of multiple interrelated ways of thinking that each help you become a better programmer.

#### Logical Reasoning

This is defined as the ability to analyse a program to predict its behavior or output. It implies things like:

-	Being able to trace your code (<a href="/#/blogs/tracing_your_code">click here for further information</a>)
-	Understanding the syntax of a block of code
-	Understanding the flow of a block of code

You must be able to get <span class="bold">a general idea of what a piece of code</span> does without running it. Though looking at code might feel like trying to read gibberish right now, it won’t always be this way as you keep on learning.

#### Algorithmic Thinking

You wil be able to create step-by-step instructions and rules for solving problems. This involves things such as:

-	Analysing the prompt of a coding problem (<a href="/#/blogs/tracing_your_code#original-prompt">more information here</a>)
-	Planning what steps need to be taken by the machine to solve the problem
-	Using tools like flowcharts or pseudocode to plan out a program
-	Being able to turn these flowcharts/pseudocode into actual code

When tasked with creating the solution to a problem, you must be able to <span class="bold">turn instructions from English to machine-readable code</span>.

#### Abstraction

Abstraction means being able to remove unnecessary details and identifying general principles of programming. Abstraction is made up of things like:

-	Understanding the meaning and usage of programming concepts such as variables, functions, objects, classes
-	Knowing where to apply those concepts

Not only must you understand <span class="bold">what</span> those concepts mean, but you must also know <span class="bold">where</span> to use them. What use would a chef be if he knows what salt is, but not that it should be put into a soup?

#### Pattern Recognition

This sub-domain is mainly about observing and finding similarities and patterns in data. In your case, it involves things such as:

-	Identifying repetitive code
-	Analysing data and finding similarities within it (<a href="/#/blogs/use_codingcat_efficiently">like the test cases given to you on CodingCat!</a>)

A big part of programming is being able to <span class="bold">make generalizations</span>. Recognizing patterns in code means you'll be able to tell more easily what a piece of code does.

#### Decomposition

Decomposition is to break down a programming project or problem into small, manageable parts. You can assign timelines and roles, and create steps to be done.

<small>
For now, your projects are still small enough for you to not have to do much decomposition, but you’re going to work on things like that further down the line.
</small>

#### Evaluation

Evaluation refers to the ability to judge whether a solution is right for a coding problem. It implies things such as:

-	Being able to tell if a piece of code has bugs in it, without running it
-	Critically evaluating whether a solution is the best fit for a given problem
-	Being able to explain a solution in English

You must be able to <span class="bold">tell when your code needs improvement</span>. If you're blind to that, you won't be able to get better at coding easily.

### Conclusion

Three years ago, I was in your place, a first-semester Computer Science student. Now, I’ve gotten much better in all these components of computational thinking despite putting in no conscious effort.

Imagine how much further you’ll get now that you’re aware of the impact of computational thinking! One day, you’ll be looking back at all the work you put in, and you’ll be able to enjoy the progress you’ve made without even realizing it.
